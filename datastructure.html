1. Classifications of Data Structures

Primitive vs Non-Primitive

  • Primitive: Built directly into the programming language. They hold a single value.
    Examples: int, char, float, bool
  • Non-Primitive: Derived from primitive types. They can hold multiple values either of the same type or different types.
    Examples: Arrays, Linked Lists, Trees

Linear vs Non-Linear

  • Linear: Elements are arranged sequentially. Each element is attached to its previous and next adjacent elements.
    Examples: Array, Stack, Queue, Linked List
  • Non-Linear: Elements are arranged hierarchically or interconnected randomly. An element can connect to multiple other elements.
    Examples: Trees, Graphs

2. ADT, Dynamic Structures, and DMA

  • Abstract Data Type (ADT): An ADT is a theoretical concept. It defines the logical behavior of a data structure (what operations it can perform) without specifying how those operations will be implemented in code. (e.g., Stack ADT defines push/pop, but doesn't care if you build it with an array or a linked list).
  • Dynamic Data Structures: Data structures whose memory size can expand or shrink during the execution (runtime) of the program. (e.g., Linked Lists).
  • Dynamic Memory Allocation (DMA): The process of assigning memory space during program execution. In C, this is done using functions like malloc(), calloc(), and freed using free().

3. Arrays

Definition: An array is a linear collection of elements of the same data type stored in contiguous memory locations.

Disadvantages of Linear Arrays:

  • Fixed Size: Size must be defined at compile time and cannot be altered.
  • Memory Wastage: If allocated size is much larger than used size, memory is wasted.
  • Inefficient Operations: Insertion and deletion in the middle requires shifting all subsequent elements, taking $O(n)$ time.

4. Sparse Matrix

Definition: A matrix in which the majority of the elements are zero.

Limitations of normal storage:

Storing it as a standard 2D array wastes enormous amounts of memory on zeros and increases computation time for matrix operations.

Practical Usage:

Used heavily in Machine Learning, Graph algorithms (Adjacency matrices), and scientific simulations. To save space, they are stored using Triplets (Row, Column, Value) or formats like CSR (Compressed Sparse Row).

5. Address Calculation (Row-Major Order)

General Formula (0-based indexing assumed for C/C++):

$$\text{Address}(A[i][j]) = \text{Base} + W \times [ (i \times N) + j ]$$

Where $W$ = element size, $N$ = total columns, $i$ = row index, $j$ = col index.

Part A: Address of $A[2][1]$

Note: Problem statement does not provide total columns ($N$). Formula written algebraically.

$$\text{Loc} = 1000 + 2 \times (2 \times N + 1)$$

$$\text{Loc} = 1000 + 4N + 2$$

Part B: For a 3 x 4 Matrix, find $A[2][3]$

Given: Base = 1000, W = 2, Rows($M$) = 3, Cols($N$) = 4.

Target: $i = 2, j = 3$.

$$\text{Loc} = 1000 + 2 \times (2 \times 4 + 3)$$

$$\text{Loc} = 1000 + 2 \times (8 + 3)$$

$$\text{Loc} = 1000 + 22 = \mathbf{1022}$$

6. Circular Linked List

Definition: A variation of a linked list where the next pointer of the last node points back to the first node, forming a closed circle instead of pointing to NULL.

Primary Advantage: We can traverse the entire list starting from any arbitrary node. It is highly useful in systems requiring round-robin scheduling (like CPU time sharing) where the sequence needs to repeat continuously.

7, 8, 9. The Stack Data Structure

Definition: A linear data structure in which insertions and deletions are strictly allowed at one end only, called the Top.

LIFO: It stands for Last-In, First-Out. The element that is pushed last is the very first one to be popped off (like a stack of dinner plates).

Applications of Stack:

  • Recursion: The OS uses a call stack to store function states and return addresses during recursive calls.
  • Expression evaluation and conversion (Infix to Postfix).
  • Undo/Redo mechanisms in text editors.
  • Parentheses matching in compilers.

Overflow & Underflow:

  • Overflow: Occurs when trying to Push an element into a stack that is completely full (if Top == MAX - 1).
  • Underflow: Occurs when trying to Pop an element from an empty stack (if Top == -1).

C Functions for Push and Pop (Array Implementation)

*Note: We pass 'top' as a pointer (*top) so its value updates globally.*

// PUSH Function in C

void push(int stack[], int *top, int MAX, int item) {
    if (*top == MAX - 1) {
        printf("Stack Overflow\n");
        return;
    }
    (*top)++;
    stack[*top] = item;
    printf("Item %d Pushed\n", item);
}

// POP Function in C

int pop(int stack[], int *top) {
    if (*top == -1) {
        printf("Stack Underflow\n");
        return -1; // Return error value
    }
    int item = stack[*top];
    (*top)--;
    return item;
}

10, 11, 12, 13. The Queue Data Structure

Stack vs Queue

Feature Stack Queue
Principle LIFO FIFO
Ends used One end (Top) Two ends (Front, Rear)
Operations Push / Pop Enqueue / Dequeue

Queue & FIFO

Definition: A linear data structure where insertions occur at one end (Rear) and deletions occur at the other end (Front).

FIFO: First-In, First-Out. The element inserted first is the first one to be removed. It perfectly models real-world scenarios like a line at a ticket counter or printer job scheduling.

Circular Queue & Advantages

Definition: A queue where the last position is connected back to the first position, forming a circle. (Rear = (Rear + 1) % MAX).

Advantage over Linear Queue:
In a linear queue, if the Rear reaches the maximum size, we cannot insert new elements even if the Front has moved forward creating empty spaces. A Circular Queue reuses these empty spaces efficiently.

// ENQUEUE & DEQUEUE in C

void enqueue(int Q[], int *front, int *rear, int MAX, int item) {
    if (*rear == MAX - 1) {
        printf("Queue Full\n");
        return;
    }
    if (*front == -1) *front = 0;
    
    (*rear)++;
    Q[*rear] = item;
}

int dequeue(int Q[], int *front, int *rear) {
    if (*front == -1 || *front > *rear) {
        printf("Queue Empty\n");
        return -1;
    }
    int item = Q[*front];
    (*front)++;
    return item;
}

16, 17. Recursion vs Iteration

Basis Recursion Iteration
Definition Function calls itself continuously until a base case is met. A set of instructions repeatedly executed via loops (for/while).
Memory High memory overhead (uses system Call Stack). Low memory overhead ($O(1)$ extra space).
Speed Slower due to overhead of function calls. Faster execution.

1. Factorial (Recursive C)

int fact_rec(int n) {
    if (n == 0 || n == 1)
        return 1;
    else
        return n * fact_rec(n - 1);
}

2. Factorial (Iterative C)

int fact_iter(int n) {
    int f = 1;
    for(int i = 1; i <= n; i++) {
        f = f * i;
    }
    return f;
}

3. Nth Fibonacci (Recursive C)

int fib(int n) {
    if (n <= 1)
        return n;
    else
        return fib(n-1) + fib(n-2);
}

18, 19. Linear and Binary Search

Linear Search vs Binary Search

Prerequisite for Binary Search: The array MUST be sorted.

  • Linear Search: Compares target sequentially with each element. Time Complexity: $O(N)$. Works on unsorted data.
  • Binary Search: Divide and conquer approach. Checks middle element, discards half of the array. Time Complexity: $O(\log N)$. Extremely fast for large datasets.

Linear Search in C

int linearSearch(int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i; // Found at index i
        }
    }
    return -1; // Not Found
}

Binary Search in C (Iterative vs Recursive)

Comparison: The iterative approach uses $O(1)$ extra space, making it generally preferred. The recursive approach takes $O(\log N)$ auxiliary space due to the call stack.

Iterative Binary Search

int binarySearchIter(int arr[], int n, int target) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        
        if (arr[mid] == target)
            return mid;
        else if (arr[mid] < target)
            low = mid + 1;
        else
            high = mid - 1;
    }
    return -1;
}

Recursive Binary Search

int binarySearchRec(int arr[], int low, int high, int target) {
    if (low > high)
        return -1;

    int mid = low + (high - low) / 2;
    
    if (arr[mid] == target)
        return mid;
    else if (arr[mid] < target)
        return binarySearchRec(arr, mid + 1, high, target);
    else
        return binarySearchRec(arr, low, mid - 1, target);
}
